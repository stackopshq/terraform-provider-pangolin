# Pangolin Provider

The **Pangolin** provider allows you to manage resources in [Pangolin](https://github.com/fosrl/pangolin), an open-source self-hosted tunneling and reverse proxy solution.

Use this provider to automate the full lifecycle of your Pangolin infrastructure via Infrastructure as Code: organizations, sites, HTTP resources, access control, authentication, identity providers, roles, users, and API keys.

## Example Usage

```terraform
terraform {
  required_providers {
    pangolin = {
      source  = "stackopshq/pangolin"
      version = "~> 1.3"
    }
  }
}

provider "pangolin" {
  url     = "https://pangolin.example.com"
  api_key = var.pangolin_api_key
  org_id  = "my-org"
}

# Create a site (tunnel connector)
resource "pangolin_site" "main" {
  name = "prod-site"
  type = "newt"
}

# Create an HTTP resource
resource "pangolin_resource" "app" {
  name      = "my-app"
  subdomain = "app"
  domain_id = "domain1"
}

# Attach a backend target
resource "pangolin_target" "app" {
  resource_id = pangolin_resource.app.id
  ip          = "192.168.1.10"
  port        = 8080
}

# Protect with a password
resource "pangolin_resource_password" "app" {
  resource_id = pangolin_resource.app.id
  password    = var.app_password
}

# Add an access rule
resource "pangolin_resource_rule" "allow_internal" {
  resource_id = pangolin_resource.app.id
  action      = "ACCEPT"
  match       = "CIDR"
  value       = "10.0.0.0/8"
  priority    = 1
}
```

## Schema

### Optional

- `api_key` (String, Sensitive) The API key for authentication. Can be set via `PANGOLIN_API_KEY` env var.
- `org_id` (String) The organization ID. Can be set via `PANGOLIN_ORG_ID` env var.
- `url` (String) The base URL of the Pangolin API (e.g. `https://pangolin.example.com`). Can be set via `PANGOLIN_URL` env var.

## Authentication

The provider requires an API key with appropriate permissions. You can create one via the Pangolin web UI or using the `pangolin_api_key` resource.

All configuration can also be provided via environment variables:

```sh
export PANGOLIN_URL="https://pangolin.example.com"
export PANGOLIN_API_KEY="your-api-key"
export PANGOLIN_ORG_ID="your-org-id"
```
